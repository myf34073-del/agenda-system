# 议题一体化系统 · 后端设计文档

## 快速启动

1. 双击运行 `install_deps.bat`（首次安装，约30秒）
2. 双击运行 `start_backend.bat`（启动服务，保持窗口开启）
3. 用浏览器打开 `miniapp_preview.html`
4. 在弹出的登录框输入账号密码，开始使用

---

## 系统架构

```
miniapp_preview.html (前端界面)
        ↕ HTTP API (CORS)
Flask 后端 (localhost:5001)
        ↕
SQLite 数据库 (backend/agendas.db)
```

## API 接口清单

### 认证
- `POST /api/auth/login` — 登录（返回token）
- `POST /api/auth/logout` — 退出
- `GET  /api/auth/me` — 获取当前用户

### 用户管理（admin）
- `GET  /api/users` — 用户列表
- `POST /api/users` — 新建用户
- `PUT  /api/users/:id` — 修改用户

### 议题模板
- `GET /api/templates` — 获取模板列表（按部门过滤）
- `GET /api/templates/:id` — 获取单个模板

### 议题管理（核心）
- `GET    /api/agendas` — 议题列表（支持分页/搜索/筛选）
- `POST   /api/agendas` — 新建议题
- `GET    /api/agendas/:id` — 议题详情（含审批记录）
- `PUT    /api/agendas/:id` — 编辑议题
- `DELETE /api/agendas/:id` — 废止议题
- `POST   /api/agendas/:id/action` — 审批动作（提交/通过/驳回）

### 议题流转动作（action参数）
| action | 说明 | 当前状态 → 新状态 |
|--------|------|-----------------|
| submit | 提交审核 | draft → submitted |
| dept_pass | 科室通过 | submitted → dept_review |
| dept_reject | 科室驳回 | submitted → rejected |
| office_pass | 局办初审通过 | dept_review → office_review |
| office_reject | 局办初审驳回 | dept_review → rejected |
| schedule | 排期上会 | office_review → scheduled |
| meeting_pass | 上会审议通过 | scheduled → meeting |
| minutes_draft | 纪要已撰写 | meeting → minutes_draft |
| sign | 签发 | minutes_draft → minutes_signed |
| archive | 归档 | minutes_signed → archived |

### 会议管理
- `GET  /api/meetings` — 会议批次列表
- `POST /api/meetings` — 创建新批次
- `GET  /api/meetings/:id` — 会议详情（含上会议题）
- `POST /api/meetings/:id/agendas` — 添加议题到会议

### 会议纪要
- `GET  /api/minutes` — 纪要列表
- `POST /api/minutes` — 新建纪要
- `PUT  /api/minutes/:id` — 修改/签发纪要

### 议题转换（场馆中心）
- `GET  /api/conversions` — 转换记录列表
- `POST /api/conversions` — 发起转换请求（系统自动给出建议）
- `PUT  /api/conversions/:id/confirm` — 确认转换，生成新议题

### 通知
- `GET  /api/notifications` — 通知列表（含未读数）
- `POST /api/notifications/read-all` — 全部已读

### 统计
- `GET /api/stats/overview` — 数据概览（收支/部门排行/三重一大）

## 前端集成说明

前端通过 `X-Token` 请求头传递登录Token（存储在 `localStorage`）。

**自动化行为：**
- 进入"议题库"页面 → 自动从数据库加载真实议题列表
- 进入"数据统计"页面 → 自动从数据库加载收支和三重一大数据
- 写议题表单点"提交" → 议题保存到数据库 + 推送审批通知
- 页面加载时检测登录状态，未登录自动弹出登录框
